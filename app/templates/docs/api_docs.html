1. What is this system? 🤔

Imagine you have a lemonade stand online. When someone wants to buy a glass of lemonade, they need a way to pay you. Vitta Setu is like a magical bridge between your lemonade stand and the customer's phone.

You tell Vitta Setu: "Hey, charge this person ₹50 for lemonade." Vitta Setu creates a special payment webpage for them. Once they pay, Vitta Setu runs back to you and says: "They paid!" so you can give them the lemonade.

2. The Secret Password 🗝️ (Authentication)

Before Vitta Setu talks to your lemonade stand, it needs to know it's really you. We use a secret password called an API Key. You must show this key every time you ask Vitta Setu to do something.

You put this key in the Authorization header of your request, like putting a secret stamp on an envelope.

Authorization: Bearer pk_test_your_public_key_here

3. Creating a Payment 💸

To make a payment page, you need to send a message (a POST Request) to this address: /api/v1/create-payment

What you need to send to us (The Request):
NameRequired?What does it mean?
amountYes ✅How much money? (e.g. 50.0)
upi_addressYes ✅Where should the money go? (e.g. yourname@bank)
target_callback_urlYes ✅The phone number (URL) we should call when they finish paying to tell you the result.
return_urlYes ✅Where should we send the customer after they pay? (e.g. back to your lemonade stand website)
merchant_transaction_idNo ❌Your special receipt number to remember this order.
What we will send back to you (The Response):

We will give you a magical link (payment_url). You just send your customer to this link so they can pay!

{
  "status": "success",
  "session_id": "uuid-v4",
  "payment_url": "https://vittasetu.com/payment/token123",
  "expiry_time": "2023-10-01T12:00:00Z"
}

4. Getting Notified 📬 (Webhooks)

Remember the target_callback_url you gave us earlier? When the customer finishes paying, our system will automatically "call" that URL to tell your server what happened (Success, Failed, or Unknown). This is called a Webhook.

But wait! How do you know the call is really from Vitta Setu and not a bad guy pretending to be us? We use a super-secret mathematical signature called HMAC.

Checking the Signature (Python Example)

You look at the secret X-VS-Signature we send, and do some math to see if it matches your secret key. Here is how you do it in Python:

import hmac, hashlib, json

def verify_webhook(secret, payload_dict, timestamp, received_signature):
    # We turn the message into a tight string
    message = f"{timestamp}.{json.dumps(payload_dict, separators=(',', ':'))}"
    
    # We calculate our own signature using our secret key
    expected_signature = hmac.new(
        secret.encode('utf-8'),
        message.encode('utf-8'),
        hashlib.sha256
    ).hexdigest()
    
    # If they match, it's really from Vitta Setu!
    return hmac.compare_digest(expected_signature, received_signature)
